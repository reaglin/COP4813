Styling Web Pages with CSS — Module Topics

Introduction to CSS

An overview of what Cascading Style Sheets are and how they work alongside HTML to control the visual presentation of web pages. Students learn how to link CSS to their HTML documents using inline, internal, and external stylesheet methods.
  • What is CSS? — CSS, or Cascading Style Sheets, is a stylesheet language used to control the visual presentation of HTML documents.
  • How CSS Works with HTML — CSS works alongside HTML by targeting HTML elements and applying visual rules to them.
  • Inline CSS — Inline CSS is applied directly to a single HTML element using the style attribute within the opening tag.
  • Internal CSS — Internal CSS is written inside a <style> element placed within the <head> section of an HTML document.
  • External CSS — External CSS stores all styles in a separate .css file that is linked to one or more HTML documents.
  • Choosing the Right Method — Each of the three CSS methods — inline, internal, and external — has appropriate use cases depending on the scope and scale of styling needed.

CSS Selectors

Exploration of the different types of CSS selectors including element, class, ID, and combination selectors used to target specific HTML elements. Students learn how specificity and the cascade determine which styles are applied when rules conflict.
  • Element Selectors — Element selectors target HTML elements directly by their tag name, applying styles to every instance of that element on the page.
  • Class Selectors — Class selectors target elements that share a specific class attribute, allowing the same styles to be applied to multiple elements regardless of their tag type.
  • ID Selectors — ID selectors target a single, unique element on a page using the element's id attribute, making them ideal for styling one-of-a-kind components.
  • Combination Selectors — Combination selectors allow developers to target elements based on their relationship to other elements or by combining multiple conditions, enabling precise styling control.
  • CSS Specificity — Specificity is a scoring system browsers use to determine which CSS rule takes precedence when multiple rules target the same element and property.
  • The Cascade — The cascade is the algorithm CSS uses to resolve conflicts between rules, considering origin, specificity, and source order to decide which styles are ultimately applied.

The Box Model

A foundational concept explaining how every HTML element is treated as a rectangular box with content, padding, border, and margin areas. Students learn how to manipulate these properties to control spacing and sizing of elements on a page.
  • What Is the Box Model? — Every HTML element on a web page is treated by the browser as a rectangular box. This box is made up of four distinct areas that together form the CSS Box Model.
  • The Content Area — The content area is the innermost part of the box, where the actual text, images, or other media of an element are displayed.
  • Padding — Padding is the space between the content area and the element's border. It creates breathing room inside the element, pushing the content away from the edges.
  • Border — The border wraps around the padding and content areas, forming a visible or invisible boundary around the element. It sits between the padding and the margin.
  • Margin — Margin is the outermost layer of the box model, creating space outside the element's border to separate it from neighboring elements.
  • The box-sizing Property — By default, the 'width' and 'height' properties in CSS apply only to the content area, meaning padding and border are added on top of that size. The 'box-sizing' property changes how the total size of an element is calculated.
  • Inspecting the Box Model with DevTools — Modern browser developer tools provide a visual diagram of the box model for any selected element, making it easy to inspect and debug spacing and sizing issues.

Typography and Colors

Covers CSS properties for styling text including font family, size, weight, line height, and text alignment to improve readability. Students also learn how to apply colors using named values, hex codes, and RGB notation to both text and backgrounds.
  • Font Family — The font-family property specifies the typeface used to display text on a web page.
  • Font Size and Weight — Font size controls how large text appears, while font weight controls its thickness or boldness.
  • Line Height and Text Alignment — Line height and text alignment are key properties that directly affect the readability and layout of text blocks.
  • Applying Colors with Named Values and Hex Codes — CSS allows colors to be specified using human-readable named values or precise hexadecimal codes.
  • Applying Colors with RGB Notation — RGB notation defines colors by specifying the intensity of red, green, and blue channels as numeric values from 0 to 255.
  • Styling Text Color and Background Color — CSS provides two distinct properties for applying color: one for text and one for the background of an element.

CSS Layout Techniques

An introduction to modern CSS layout methods including Flexbox and basic positioning to arrange elements on a web page. Students apply these techniques to structure multi-section page layouts in a visually organized way.
  • Understanding CSS Layout Flow — By default, browsers arrange HTML elements using normal document flow, where block elements stack vertically and inline elements sit side by side.
  • Introducing Flexbox — Flexbox is a one-dimensional CSS layout model that arranges items in a row or column, making it easy to align and distribute space among elements in a container.
  • Aligning and Justifying Flex Items — Flexbox provides two primary axes — the main axis and the cross axis — each controlled by dedicated alignment properties.
  • Controlling Individual Flex Items — Properties applied directly to flex children let you override container-level alignment and control how each item grows or shrinks.
  • CSS Positioning — The position property removes elements from normal flow or adjusts their placement relative to a reference point, enabling overlapping and fixed UI elements.
  • Structuring Multi-Section Page Layouts — Combining Flexbox and positioning allows you to build organized, multi-section page layouts such as headers, sidebars, main content areas, and footers.

Styling a Complete Web Page

Students bring together all learned CSS concepts to apply cohesive visual styling to the websites they built in the previous lesson. This capstone topic reinforces best practices for writing clean, organized CSS stylesheets.
  • Linking Your Stylesheet to Your HTML Page — Before any CSS can be applied, the external stylesheet must be properly connected to the HTML document using a link element in the head section.
  • Organizing Your CSS Stylesheet — A well-organized stylesheet groups related rules together and uses comments to label each section, making the code easier to read and maintain.
  • Applying Typography Across the Page — Consistent typography ties a web page together visually by establishing a clear hierarchy of headings, body text, and supporting content.
  • Using Color Cohesively — A cohesive color scheme unifies all sections of a web page and reinforces the overall visual identity of the site.
  • Styling Layout and Spacing with the Box Model — Proper use of margin, padding, and width controls how each element occupies space on the page and how sections relate to one another visually.
  • Styling Navigation and Interactive Elements — Navigation links and buttons are the primary interactive elements on most pages and should receive deliberate styling that signals their purpose to users.
  • Reviewing and Refining the Finished Stylesheet — Once all styles are applied, a final review pass ensures the stylesheet is clean, consistent, and free of redundant or conflicting rules.
